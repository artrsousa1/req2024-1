    Requisitos elicitados

    Esta seção é destinada para a elicitação dos requisitos de todas as áreas definidas da Play Store, e a sua devida priorização para as etapas posteriores neste projeto, além disto esta secção esta dividida em requisitos funcionais (o que um sistema deve fazer) e não funcionais (como o sistema deve operar).

    Requistos Funcionais

    Requisito Definição Técnica de elicitação Prioridade
    REQ01 O usuário deve poder buscar por um app ou jogo Introspecção, Análise de protocolo Must
    REQ02 O usuário deve poder instalar um app ou jogo Introspecção, Análise de protocolo Must
    REQ03 O usuário deve poder atualizar um app ou jogo Introspecção, Observação Must
    REQ04 O usuário deve poder desinstalar um app ou jogo Introspecção, Análise de protocolo Must
    REQ05 O sistema deve fornecer a política de privacidade de um app ou jogo Introspecção, Análise de protocolo Must
    REQ06 O sistema deve mostrar em quais sistemas operacionais um app ou jogo pode ser executado Análise de protocolo Must
    REQ07 O sistema deve mostrar qual é a data de lançamento de um app ou jogo Análise de protocolo Must
    REQ08 O sistema deve mostrar quais são os dados que um app ou jogo extrai dos usuários Introspecção, Análise de protocolo Must
    REQ09 O sistema deve mostrar quais são as permissões que um app ou jogo tem na máquina do usuário Análise de protocolo Must
    REQ10 O sistema deve mostrar o que um app ou jogo faz com os dados extraídos dos seus usuários Introspecção, Análise de protocolo Must
    REQ11 O usuário deve poder pedir a deleção dos dados coletados por um jogo ou app Análise de protocolo Must

    REQ30 O sistema deve mostrar qual a empresa criadora do app ou jogo Análise de protocolo Should
    REQ31 O aplicativo deve fornecer acesso à funcionalidade "esqueci a senha" do Gmail. Observação, Introspecção Should

    REQ55 O sistema deve persistir as pesquisas mais recentes de um usuário Introspecção Would
    REQ56 O sistema deve fornecer um tutorial de como o app ou jogo funciona Análise de protocolo Would
    REQ57 O sistema deve mostrar a quantidade de downloads que um app ou jogo tem Introspecção, Análise de protocolo Would
    REQ58 O aplicativo deve permitir configurações personalizadas para cada conta. Observação, Introspecção Would
    REQ59 O aplicativo deve gerenciar os benefícios e ferramentas disponíveis para a Play Store vinculadas ao Gmail. Observação, Introspecção Would
    REQ60 O Sistema deve ser capaz de mostrar os apps disponíveis para um dispositivo Análise de protocolo Would
    REQ61 O Sistema deve ser capaz de instalar apps em outros dispositivos Análise de protocolo Would
    REQ62 O Sistema deve categorizar os aplicativos por idade na área kids Introspecção Would
    REQ63 O sistema deve fornecer feedbacks para o usuário, como mensagens de erro. Brainstorm Should
    REQ64 O sistema deve mostrar a localização do usuário dentro do app. Brainstorm Would
    REQ65 O sistema deve mostrar botões para o usuário confirmar suas ações, evitando erros. Brainstorm Should
    REQ66 O sistema deve permitir a mudança de idioma. Brainstorm Should
    REQ67 O sistema deve ter um menu de ajuda. Brainstorm Could
    REQ68 O sistema deve permitir a busca de apps/jogos por voz. Brainstorm Could

    Requisitos não funcionais

    Código Descrição Técnica de elicitação Prioridade
    RNF01 O Sistema deve ter uma hierarquia lógica de estruturação. Brainstorm, NFR Should
    RNF02 O Sistema deve ser acessível para pessoas com deficiências. Brainstorm, NFR Could
    RNF03 O Sistema deve ser acessível para pessoas idosas. Brainstorm, Storytelling App, Storytelling fluxo de usuário Could
    RNF04 O Sistema deve ser consistente e padronizado, de acordo com as heurísticas de Nielsen. Brainstorm, NFR Could
    RNF05 O Sistema deve permitir descrição por áudio e atalhos de teclado. Brainstorm, NFR Could
    RNF06 O Sistema deve ter cores contrastantes. Brainstorm, NFR Should
    RNF07 O sistema deve permitir configurações de acessibilidade como cor e fonte. Brainstorm, NFR Could
    RNF08 O sistema deve se adaptar a vários tipos de telas. Brainstorm, NFR Should
    RNF09 O sistema deve permitir a recuperação de falhas. Brainstorm, NFR Must
    RNF10 O sistema deve garantir que os dados dos aplicativos e dos usuários sejam validados e armazenados com precisão para evitar corrupção ou perda de dados. Brainstorm, NFR Must
    RNF11 O sistema deve prevenir erros. Brainstorm, NFR Must
    RNF12 O sistema deve ter tolerência para com falhas. Brainstorm, NFR Must
    RNF13 O sistema deve ficar no ar 24/7. Brainstorm, NFR Must
    RNF14 O sistema deve realizar atualizações regulares e manutenção para corrigir bugs, melhorar a performance e manter a confiabilidade geral da plataforma. Brainstorm, NFR Must
    RNF15 O sistema deve ser eficaz em diversos tipos de rede. Brainstorm, NFR Should
    RNF16 O sistema deve lidar com picos de tráfego e alta demanda sem perder a disponibilidade. Brainstorm, NFR Should
    RNF17 O sistema deve retornar o objeto de pesquisa rapidamente. Brainstorm, NFR Should
    RNF18 O sistema deve garantir que as transações de compra e download de aplicativos sejam processadas rapidamente. Brainstorm, NFR Could
    RNF19 O sistema deve otimizar o tempo de download e instalação de aplicativos. Brainstorm, NFR Could
    RNF20 O sistema deve otimizar o uso dos recursos (rede, bateria e memória) do dispositivo. Brainstorm, NFR Could
    RNF21 O sistema deve funcionar em qualquer dispositivo Android. Brainstorm, NFR Should
    RNF22 O sistema deve funcionar bem em diversos navegadores. Brainstorm, NFR Should
    RNF23 O sistema deve assegurar que a Play Store funcione de maneira consistente em diferentes dispositivos e sistemas operacionais. Brainstorm, NFR Should
    RNF24 O sistema deve ter medidas de segurança para proteger a plataforma contra ataques cibernéticos, malware e outras ameaças. Brainstorm, NFR Must
    RNF25 O sistema deve garantir que os dados sensíveis do usuário sejam criptografados. Brainstorm, NFR Must
    RNF26 O sistema deve garantir que as políticas de privacidade da Play Store e dos apps sejam disponibilizadas. Brainstorm, NFR Must
    RNF27 O sistema deve informar os dados compartilhados com um app. Brainstorm, NFR Must
    RNF28 O sistema deve seguir a LGPD. Brainstorm, NFR Must
